StyleCast ERP Backend — System Design Document
1. Overview

The StyleCast ERP Backend is a multi-tenant backend system designed to support fashion brands in managing their operational workflows. The platform enables brands to manage their product catalog, track inventory, process customer orders, configure shipping rules, and view business analytics.

The backend is implemented using FastAPI and PostgreSQL, with SQLAlchemy ORM for database interaction and Pydantic for request and response validation.

Each brand is treated as a tenant within the system, meaning their data is logically isolated using a brand_id field across core entities.

The system exposes RESTful APIs that can be consumed by a future admin dashboard or marketplace platform.

5. Multi-Tenant Architecture

The platform supports multiple brands using a shared database.

Tenant isolation is implemented through the brand_id field across core business entities.

Example tables containing tenant scope:

products

variants

inventory

orders

shipping_rules

In production environments, API queries would always be scoped by the authenticated user's brand.

Example:

db.query(Product).filter(Product.brand_id == current_user.brand_id)

This ensures brands cannot access another brand’s data.

7. Order Processing Flow

Order creation follows this process:

Client submits order request

Backend validates the product variant

Inventory stock availability is checked

Inventory quantity is reduced

Order record is created

Order items are stored

Response returned to client

This workflow ensures accurate inventory management and prevents overselling.
